 51  Navigating Graphs 
 52  ----------------- 
 53   
 54  In addition to storing your data, Nodes and Edges have 
 55  a few other special properties by default. For Nodes, 
 56  these properties are "incoming" and "outgoing", and 
 57  they contain the incoming and outgoing edges attached 
 58  to that node. For Edges, these properties are (again, 
 59  by default) "start" and "end". 
 60   
 61  To get all the outgoing edges of a particular node: 
 62   
 63          >>> n1.outgoing 
 64          [Edge(weight=5)] 
 65   
 66  And to get the incoming edges: 
 67   
 68          >>> n2.incoming 
 69          [Edge(weight=5)] 
 70   
 71  You can use this to get all the nodes adjacent to 
 72  an interesting Node: 
 73   
 74          >>> adjacent = [edge.end for edge in node1.outgoing] 
 75   
 76  Or to get the degree of the given node: 
 77   
 78          >>> degree = len(node1.outgoing) + len(node2.incoming) 
 79           
 80  You can combine this with Edges' "start" and "end" properties 
 81  to navigate your graph, if you know how your data is related 
 82  structurally. For example, to get the oldest outgoing edge attached 
 83  to the oldest node attached to node_1, I could do the following: 
 84   
 85          >>> interesting_node = node_1.outgoing[0].end.outgoing[0] 
 86   
 87  Of course, there are scenarios in which you don't know how to 
 88  navigate your graph, or need to find a starting point that isn't 
 89  easily determined based on a given element's properties. For that, 
 90  Graph supports Node and Edge iteration for the simple cases, and 
 91  traversals for more complex behavior. 
 92   
 93  To iterate over all the nodes in a graph: 
 94   
 95          >>> for node in g.nodes: 
 96          >>>     print(node) 
 97          Node(name="bob") 
 98          Node(weight=5) 
 99   
100  To do the same for edges: 
101   
102          >>> for edge in g.edges: 
103          >>>     print(edge) 
104          Edge() 
105          Edge(weight=5) 
106   
107  If you only want certain nodes, the search 
108  functions are provided for convenience: 
109   
110          >>> for node in g.search_nodes(name="bob"): 
111          >>>     print(node) 
112          Node(name="bob") 
113           
114  And for edges: 
115   
116          >>> for edge in g.search_edges(weight=5): 
117          >>>     print(edge) 
118          Edge(weight=5) 
119   
120  You can also search for elements which demonstrate data 
121  equality with a given element. This is often useful when 
122  comparing elements from different graphs. 
123   
124  To get all equivalent nodes: 
125   
126          >>> n = Node(name="bob") 
127          >>> g.get_equivalent_nodes(n) 
128          {Node(name="bob")} 
129   
130  And its edge-comparing equivalent 
131   
132          >>> e = Edge(node_2, node_3, weight=5) 
133          >>> g.get_equivalent_edges(e) 
134          {Node(weight=5)} 
135   
136  Of course, the edge returned by this function *is* datawise 
137  equivalent to the edge we just created. But more likely, 
138  when you're looking at edges you're interested in the nodes 
139  around them as much as you are the node itself. In the 
140  example above, the edge returned is attached at different 
141  points than the edge given. To solve this, you can use the  
142  optional "flatten" argument, which causes it to also compare 
143  the data in edges incident to the node given or nodes  
144  incident to the edge given. 
145   
146  Three traversals are provided by default- A*, depth first, 
147  and breadth first. 
148   
149  To do a depth first traversal: 
150   
151          >>> for node in g.depth_first_traversal(node_1): 
152          >>>     print(node) 
153          Node(name="bob") 
154          Node(weight=5) 
155   
156  Depth first traversals are similar in usage, simply 
157  substituting "depth_first_traversal" for "breadth_first_traversal". 
158   
159  A* traversals require an additional callable argument. 
160  This callable will be passed a list of unvisited nodes 
161  and should select and remove exactly one of them to return. 
162   
163  For example, to navigate a graph by visiting the most 
164  popular (ie, most incoming edges) nodes first, I could 
165  write a getter function as follows: 
166   
167          >>> def get_popularity(node): 
168          >>>     return len(node.incoming) 
169   
170  A selector function: 
171   
172          >>> def get_most_popular(nodes): 
173          >>>     nodes.sort(nodes, key=get_popularity) 
174          >>>     return nodes.pop() 
175   
176  And traverse the graph: 
177   
178          >>> for node in g.a_star_traversal(node_1, get_heaviest): 
179          >>>     print(node) 
180          Node(weight=5) 
181          Node(name="bob") 
182   
183  Binary Graph Operations 
184  ----------------------- 
185   
186  Four basic operations are provided for the comparison 
187  of graphs:  
188   
189          1. union (|), which creates a new graph containing all 
190             the nodes and edges of its parents, 
191   
192          2. intersection (&), which creates a new graph containing 
193             all the nodes and edges not in both 
194   
195          3. difference (-), which creates a new graph containing 
196             all the nodes and edges in the first parent but 
197             not in the second. 
198   
199          4. merge (+), which creates a new graph with all the  
200             data-unique nodes from both parent graphs plus all  
201             their structurally and data unique edges.
