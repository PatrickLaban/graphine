 12  This module contains the base GraphElement, Node, Edge, 
 13  and Graph implementations for Graphine, an easy-to-use, 
 14  easy-to-extend Graph library. 
 15   
 16  Graphs generated using this representation have directed  
 17  edges, the ability to represent loops and parallel edges, 
 18  and can attach arbitrary data to nodes and edges. 
 19   
 20  Interface summary 
 21  ================= 
 22   
 23  Building Graphs 
 24  --------------- 
 25   
 26  To create a new Graph: 
 27   
 28          >>> from graph.base import Graph 
 29          >>> g = Graph() 
 30   
 31  To add nodes: 
 32   
 33          >>> node_1 = g.add_node(name="bob") 
 34          >>> node_2 = g.add_node(weight=5) 
 35          >>> node_3 = g.add_node(color="red", flags=["visited"]) 
 36   
 37  To add edges: 
 38   
 39          >>> edge_1 = g.add_edge(node_2, node_3) 
 40           
 41  Notice that edges can have properties as well: 
 42   
 43          >>> edge_2 = g.add_edge(node_1, node_2, weight=5) 
 44          >>> edge_3 = g.add_edge(node_3, node_1, stuff={}) 
 45           
 46  To remove nodes: 
 47   
 48          >>> g.remove_node(node_3) 
 49           
 50  And edges: 
 51   
 52          >>> g.remove_edge(edge_3) 
 53   
 54  Navigating Graphs 
 55  ----------------- 
 56   
 57  In addition to storing your data, Nodes and Edges have 
 58  a few other special properties by default. For Nodes, 
 59  these properties are "incoming" and "outgoing", and 
 60  they contain the incoming and outgoing edges attached 
 61  to that node. For Edges, these properties are (again, 
 62  by default) "start" and "end". 
 63   
 64  To get all the outgoing edges of a particular node: 
 65   
 66          >>> n1.outgoing 
 67          [Edge(weight=5)] 
 68   
 69  And to get the incoming edges: 
 70   
 71          >>> n2.incoming 
 72          [Edge(weight=5)] 
 73   
 74  You can use this to get all the nodes adjacent to 
 75  an interesting Node: 
 76   
 77          >>> adjacent = [edge.end for edge in node1.outgoing] 
 78           
 79  You can combine this with Edges' "start" and "end" properties 
 80  to navigate your graph, if you know how your data is related 
 81  structurally. For example, to get the oldest outgoing edge attached 
 82  to the oldest node attached to node_1, I could do the following: 
 83   
 84          >>> interesting_node = node_1.outgoing[0].end.outgoing[0] 
 85   
 86  Of course, there are scenarios in which you don't know how to 
 87  navigate your graph, or need to find a starting point that isn't 
 88  easily determined based on a given element's properties. For that, 
 89  Graph supports Node and Edge iteration for the simple cases, and 
 90  traversals for more complex behavior. 
 91   
 92  To iterate over all the nodes in a graph: 
 93   
 94          >>> for node in g.nodes: 
 95          >>>     print(node) 
 96          Node(name="bob") 
 97          Node(weight=5) 
 98   
 99  To do the same for edges: 
100   
101          >>> for edge in g.edges: 
102          >>>     print(edge) 
103          Edge() 
104          Edge(weight=5) 
105   
106  If you only want certain nodes, the search 
107  functions are provided for convenience: 
108   
109          >>> for node in g.search_nodes(name="bob"): 
110          >>>     print(node) 
111          Node(name="bob") 
112           
113  And for edges: 
114   
115          >>> for edge in g.search_edges(weight=5): 
116          >>>     print(edge) 
117          Edge(weight=5) 
118   
119  Three traversals are provided by default- A*, depth first, 
120  and breadth first. 
121   
122  To do a depth first traversal: 
123   
124          >>> for node in g.depth_first_traversal(node_1): 
125          >>>     print(node) 
126          Node(name="bob") 
127          Node(weight=5) 
128   
129  Depth first traversals are similar in usage, simply 
130  substituting "depth_first_traversal" for "breadth_first_traversal". 
131   
132  A* traversals require an additional callable argument. 
133  This callable will be passed a list of unvisited nodes 
134  and should select and remove exactly one of them to return. 
135   
136  For example, to navigate a graph by visiting the most 
137  popular (ie, most incoming edges) nodes first, I could 
138  write a getter function as follows: 
139   
140          >>> def get_popularity(node): 
141          >>>     return len(node.incoming) 
142   
143  A selector function: 
144   
145          >>> def get_most_popular(nodes): 
146          >>>     nodes.sort(nodes, key=get_popularity) 
147          >>>     return nodes.pop() 
148   
149  And traverse the graph: 
150   
151          >>> for node in g.a_star_traversal(node_1, get_heaviest): 
152          >>>     print(node) 
153          Node(weight=5) 
154          Node(name="bob")
